What about tracking plans with complex rules?

  • Propery names fitting regex patterns
  • Global rules that apply to all events
  • Allow freeform values for some properties

Same solution as with code - use automated testing!

Ensure known event payloads are allowed and blocked as expected after each change

Why is automated testing beneficial for tracking plans?

Tracking plans tell us:

  • Event was unplanned
  • Event property was unplanned
  • Event property was a schema violation
  • Common/global event rule was violated

Source settings tell us:

  • Block the entire event
  • Allow event but omit offending property(s)
  • Do nothing - allow entier event

When combined with the event payload, this is too many permutations for manual testing!
